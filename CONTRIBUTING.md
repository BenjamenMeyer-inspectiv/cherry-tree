This repository is meant for folks to play with and learn how to use `git cherry-picking`.

The theme for this repository is a Cherry Tree. For the most part everything should be
related to that theme. The exceptions are:

- git specific tooling documentation
- GitHub specific tooling
- documentation that cannot otherwise fit in the theme

PRs will be accepted if and only if they help meet that goal.
Examples:

- New branches with only INSTRUCTIONS.md
- Updating INSTRUCTIONS.md to fix bugs or make things easier to understand
- New tasks for demonstrating more complex operations.
- Updates to the `aspects` branch will be accepted if they support a task.
- The branch names, features, etc should continue in the theme of a Cherry Tree.

Please use a copy of the `INSTRUCTIONS.template.md` from the `.templates`  directory
when creating a new INSTRUCTIONS.md.
